# 示例：找房前的地段决策

`星合广场租房简图.svg` 是用本 skill 做「找房前决策」的产物：给定一个锚点（公司/常去的地方）和几个候选小区，用高德的**地理 + 路径 + 周边**能力，把每个候选的**通勤时间、距离、周边环境**算出来，再画成一张可对比的图。

> 它解决的是「**该去哪片找、哪个更值得看**」，不是「有哪些房、多少钱」——后者去贝壳/链家/中介。本 skill 只读公开地理数据，不碰房源库存与实时报价。

## 大致复现（命令示意）

```bash
S=scripts/amap.sh

# 1. 锚点 & 候选小区 → 坐标
"$S" geo "苏州市工业园区时代广场"          # 锚点
"$S" geo "苏州市工业园区星合广场"          # 候选

# 2. 候选 → 锚点 的通勤（按需选驾车/公交/步行）
"$S" transit "候选经纬度" "锚点经纬度" 苏州
"$S" driving "候选经纬度" "锚点经纬度"
"$S" distance "候选经纬度" "锚点经纬度" 1   # 1=驾车距离

# 3. 候选周边环境（地铁/超市/餐饮…）
"$S" around "候选经纬度" 地铁站 1500
"$S" around "候选经纬度" 超市 1000
```

把这些结果交给 agent，让它**称重你的标准**（通勤 vs 租金 vs 环境），输出排序 + 一张 SVG 对比图。租金等数字由你从平台/中介拿到后喂给它一起比。
